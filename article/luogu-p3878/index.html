题解 - [Luogu P3878] [TJOI2010]分金币

发表于 2021-11-11 更新于 2022-01-29 分类于 程序设计 ， 题解

题目链接

原始题面

题目描述

现在有 nnn 枚金币, 它们可能会有不同的价值, 第 iii 枚金币的价值为 viv_ivi​

现在要把它们分成两部分, 要求这两部分金币数目之差不超过 111, 问这样分成的两部分金币的价值之差最小是多少?

输入输出格式

输入格式

本题单个测试点内有多组测试数据

输入的第一行是一个正整数 TTT, 表示该测试点内数据组数

对于每组测试数据的格式为:

每组测试数据占两行

第一行是一个整数 nnn, 表示金币的个数

第二行有 nnn 个整数, 第 iii 个整数表示第 iii 个金币的价值 viv_ivi​

输出格式

对于每组数据输出一行一个整数表示答案

输入样例 #1

输出样例 #1

说明

数据规模与约定

  • 对 30%30\%30% 的数据, 保证 1≤vi≤10001 \leq v_i \leq 10001≤vi​≤1000
  • 对于 100%100\%100% 的数据, 保证 1≤T≤201 \leq T \leq 201≤T≤20, 1≤n≤301 \leq n \leq 301≤n≤30, 1≤vi≤2301 \leq v_i \leq 2^{30}1≤vi​≤230

解题思路

看数据范围就知道要折半或者大力模拟退火

模拟退火交了两页, 最高 10 分, 气得我直接码正解去了

今天的爆率那叫一个高啊.jpg

令 s[][] 为存所有选取情况的数组, 其中 s[i] 表示前一组金币个数为 iii 时的所有可能结果 (或者前一组金币个数比后一组多 iii 时个数)

先把前 ⌈n2⌉\lceil{n\over 2}\rceil⌈2n​⌉ 个金币的所有选取情况存起来, 然后对剩下的 n−⌈n2⌉n-\lceil{n\over 2}\rceiln−⌈2n​⌉ 个金币的所有情况在对应数组里二分即可

或者我们直接把二维数组换成一维的 std::set 数组也行

复杂度

O(n2n2)O(n2^{n\over 2})O(n22n​)
