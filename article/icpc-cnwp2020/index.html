B - 帅的研究

原始题面

Description

帅最近在研究统计, 嗯, 主要就是看看室友们的消费金额他把室友们每天的消费金额如实的统计在小本本上, 现在他看着室友们的消费记录, 他决定来整个活. 他将室友的消费记录分成很多段, 使得每段恰好有kkk天(k>0k>0k>0),如果这个室友消费记录统计的天数不是kkk的倍数, 最后一块时间长度小于kkk的时间段就被丢弃了. 帅想知道, 选择什么数字kkk可以得到最多的不同的段. 注意这里的段是可以反转的, 即, 子串 1,2,3 和 3,2,1 被认为是一样的

Input

第一行一个正整数nnn, 表示时间的天数. 第二行nnn个空格隔开的正整数 a1,a2,...ana_1,a_2,...a_na1​,a2​,...an​, 描述每天的消费金额

Output

第一行两个空格隔开的正整数, 第一个表示能获得的最大不同的段的个数, 第二个表示能获得最大值的kkk的个数. 第二行若干空格隔开的正整数kkk, 表示所有能够取得最大值的kkk, 请将kkk按照从小到大的顺序输出

1 1 1 2 2 2 3 3 3 1 2 3 3 1 2 2 1 3 3 2 1

Sample Output 1

2

Hint

对于100%100\%100%的数据, 1≤n≤2∗1051\leq n\leq 2*10^51≤n≤2∗105, 且对于所有的1≤i≤n1\leq i \leq n1≤i≤n, 有 1≤ai≤n1 \leq a_i \leq n1≤ai​≤n

解题思路

比赛时:

我(思索一番): emm这好像是关于kkk的单峰函数blablabla…

队友: emm好像是, 那就三分

我: 等下, 这个好像不是严格单调的, 我想想怎么证

然后直到比赛结束我们都没碰这题

出考场后:

旁边队: 这就是三分+hash啊blablabla…

如何看待自己嘴出正解之后把自己否了的行为.txt

用hash是显然的, 之后有两种做法

  1. 暴力枚举kkk

    对于当前的kkk, 一共有⌊nk⌋\lfloor{n\over k}\rfloor⌊kn​⌋个段, 分别正序和逆序计算hash统计个数即可

    可以用进制hash预处理前缀hash和后缀hash, 这样即可O(1)O(1)O(1)求任意段hash

    我们也可以加个剪枝: 记录最大种数SSS, 如果当前kkk对应的种数小于SSS, 则后面所有的kkk对应的种数都必然小于SSS

    时间复杂度为

    Θ(n+∑i=1n⌊ni⌋)=O(nlog⁡n)\Theta\left(n+\sum_{i=1}^n\left\lfloor{n\over i}\right\rfloor \right)=O(n\log n) Θ(n+i=1∑n​⌊in​⌋)=O(nlogn)

  2. 三分, 爬山…

    显然段的种数是kkk的单峰函数, 所以即使不用三分也可以爬山

    淦我比赛时候咋就没想起来

    也可以加个优化:

    我们知道⌊nk⌋\lfloor{n\over k}\rfloor⌊kn​⌋的取值最多只有2⌊n⌋2\lfloor\sqrt n\rfloor2⌊n​⌋种

    我们记录这2⌊n⌋2\lfloor\sqrt n\rfloor2⌊n​⌋种对应的最小原象, 再对其三分

    举个例子: n=21n=21n=21, ⌊nk⌋∈{1,2,3,4,5,7,10,21}\lfloor{n\over k}\rfloor\in\{1,2,3,4,5,7,10,21\}⌊kn​⌋∈{1,2,3,4,5,7,10,21}, 对应的最小原象构成的集合为{11,8,6,5,4,3,2,1}\{11,8,6,5,4,3,2,1\}{11,8,6,5,4,3,2,1}, 只需在该集合上找种数最大值, 最后小范围暴力枚举答案即可

解题思路

比赛时上来就交了发并查集, 然后自然没过

后来想了个假做法交了三发没过就扔了

不过在洛谷上能过就离谱

2020.11.02 upd: 出题人没写spj就更离谱

先说说正解

设数组vis[1..n], 其中vis[i]表示点i是否被标记

  1. 令vis[i]=0, ∀i∈[1,n]∩N\forall i\in[1,n]\cap\N∀i∈[1,n]∩N
  2. 从1到n枚举i
    1. 如果vis[i]==0, 则
      1. 从i+1到n枚举j
      2. 如果vis[j]==1且i,j间没有边, 则
        1. vis[i]=vis[j]=1 (因为图足够稠密, 所以这样是可行的)
        2. 跳出该循环
  3. 输出n3n\over 33n​个满足vis[i]==0的数i

然后说说我们的假做法

统计每个节点的度数, 之后按度数降序排序, 取前n3n\over 33n​个点按编号升序排序后输出

为什么是假做法呢

我们可以构造这样的图

画出来是这样的:

我们发现111和999的度是777, 777和888的度是222, 其余点的度是 666

按这样的做法就会输出

其中x∈{2,3,4,5,6}x\in\{2,3,4,5,6\}x∈{2,3,4,5,6}

但我们看图发现, 如果改成在前2n32n\over 332n​个点里随机取n3n\over33n​的点就有很大概率正确, 所以加个随机化取点多交几发就过了, 也可以加个校验多次随机化取点(Las Vegas随机)
