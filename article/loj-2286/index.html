题解 - [LOJ 2286] 「WC2017」挑战

发表于 2021-10-25 更新于 2022-02-27 分类于 程序设计 ， 题解

知名毒瘤卡常题. 别去做, 浪费时间

题目链接

原始题面

题目背景

你和同学们找了三道题目用来练习.

这次练习的目标是写出能在时间限制里通过尽量大规模数据的代码.

同学们纷纷写出了优秀的代码。现在，他们向你发起了挑战，他们对每个问题都设置了若干个测试数据，这是他们能通过的最大规模的测试数据。现在，他们想看一看你写的代码究竟能超过多少同学的代码，通过多大规模的测试数据.

本题分为 333 个任务，每个任务对应一道题和相应的若干个测试点，你需要对于每个任务，设计一个能通过尽量多测试点的程序。

题目描述

任务一

给定 nnn 个 323232 位无符号整数，将它们从小到大排序.

任务二

有 2n2n2n 个人在玩 「石头剪刀布」 游戏。他们排成两排，每排 nnn 个人。每个人在每一局游戏都使用固定策略，即对于第 i(i∈1,2)i (i \in 1, 2)i(i∈1,2) 排的第 j(0≤j<n)j (0 \leq j < n)j(0≤j<n) 个人，用一个整数 aija_{ij}aij​​​ 表示他的策略，其中 000 表示只出石头，111 表示只出剪刀，222 表示只出布.

现在有 qqq 个询问，每个询问给定三个整数 x,y,l(0≤x,y<n,1≤l≤n−max(x,y))x,y,l(0\leq x,y<n,1\leq l\leq n-max(x,y))x,y,l(0≤x,y<n,1≤l≤n−max(x,y))， 问将第一排的第 x∼x+l−1x∼x+l-1x∼x+l−1 个人和第二排的第 y∼y+l−1y∼y+l-1y∼y+l−1 个人比赛之后，第一排有多少个人会赢.

上文中 「比赛」 的意思是，对于所有整数 iii 满足 0≤i<l0\leq i<l0≤i<l,让第一排的第 x+ix+ix+i 个人和 第二排的第 y+iy+iy+i 个人进行 「石头剪刀布」 游戏.

任务三

我们称一个合法的括号串为：只由左括号和右括号构成，两种括号的数量相等， 且任意一个前缀的左括号数量不少于右括号数量的串。现在给定一个由 (，) 和? 构成的串，问有多少种不同的方案，使得将每个 ? 都替换成一个括号之后，该串变成一 个合法的括号串。两种方案不同，当且仅当至少有一个位置的 ? 被替换成了不同的括号。

输入输出格式

输入格式

此题提供了模板程序。选手可以在此基础上编写自己的程序，模板程序详见下文数据范围与提示.

第一行一个整数 task_id(1≤task_id≤3)task\_id(1\leq task\_id\leq3)task_id(1≤task_id≤3)，表示任务编号。接下来是每个具体任务的输入内容.

在输入的同一行中，相邻的两个整数会被一个空格隔开.

对于任务一：一行，两个整数 n,sn,sn,s。令 a0=next_integer(s),ai=next_integer(ai−1),1≤i<na_0=next\_integer(s),a_i=next\_integer(a_{i-1}),1\leq i<na0​=next_integer(s),ai​=next_integer(ai−1​),1≤i<n，则 a0,a1,…,an−1a_0,a_1,…,a_{n-1}a0​,a1​,…,an−1​ 即为需要排序的 nnn 个整数.

对于任务二：第一行两个整数 n,qn,qn,q。第二行一个仅包含 0,1,20, 1, 20,1,2 的长度为 nnn 的字符串，第 iii 个字符所代表的整数表示第一排第 iii 个人的策略（即 a1ia_{1i}a1i​​​）。第三行格式同第二行，表示第二排各个人的策略.

对于任务三：第一行一个整数 nnn，表示给定的串的长度。第二行一个字符串，即为给定的串。

输出格式

对于任务1：令 bbb 为已经排好序的数组，调用 output_arr(b, n * 4) 即可.

对于任务2：将每个询问的答案依次存入 323232 位无符号整数数组 bbb 中（即，存入 b0,b1,⋯,bq−1b_0,b_1,⋯,b_{q-1}b0​,b1​,⋯,bq−1​ 中），然后调用 output_arr(b, q * 4) 即可.

对于任务3：输出一个整数，表示不同的方案数除以 2322^{32}232​​ 得到的余数。

解题思路

任务一

基数排序, 桶要开小些以便卡入 L1 Cache

任务二

直接 O(nq)O(nq)O(nq) 暴力, 使用 popcount 指令和循环展开卡常

任务三

O(n2)O(n^2)O(n2) DP, 依旧是用循环展开卡常
