题解 - [CodeForces 1139 D] Steps to One (杜教筛版)

发表于 2021-04-25 更新于 2022-01-29 分类于 程序设计 ， 题解

为了补天梯赛题写的题解

题目链接

原始题面

time limit per test: 2 seconds

memory limit per test: 256 megabytes

input: standard input

output: standard output

Description

Vivek initially has an empty array aaa and some integer constant mmm

He performs the following algorithm:

  1. Select aaa random integer xxx uniformly in range from 111 to mmm and append it to the end of aaa
  2. Compute the greatest common divisor of integers in aaa
  3. In case it equals to 111, break
  4. Otherwise, return to step 111

Find the expected length of aaa. It can be shown that it can be represented as PQ{P\over Q}QP​ where PPP and QQQ are coprime integers and Q≠0(mod109+7)Q\ne 0\pmod{10^9+7}Q​=0(mod109+7). Print the value of P⋅Q−1(mod109+7)P\cdot Q^{-1}\pmod{10^9+7}P⋅Q−1(mod109+7)

Input

The first and only line contains a single integer mmm (1⩽m⩽1000001\leqslant m\leqslant 1000001⩽m⩽100000)

Output

Print a single integer — the expected length of the array aaa written as P⋅Q−1(mod109+7)P\cdot Q^{-1}\pmod{10^9+7}P⋅Q−1(mod109+7)

Note

In the first example, since Vivek can choose only integers from 111 to 111, he will have a=[1]a=[1]a=[1] after the first append operation, and after that quit the algorithm. Hence the length of aaa is always 111, so its expected value is 111 as well

In the second example, Vivek each time will append either 111 or 222, so after finishing the algorithm he will end up having some number of 222's (possibly zero), and aaa single 111 in the end. The expected length of the list is 1⋅121+2⋅122+3⋅123+...=2\def\enum#1{ #1\cdot{1\over 2^{ #1}}}\enum{1}+\enum{2}+\enum{3}+...=21⋅211​+2⋅221​+3⋅231​+...=2

题意简述

对一个初始为空的正整数序列 aaa, 每次随机在 [1,m]N[1,m]_{\N}[1,m]N​ 内选一个数插入, 当其最大公因数为 111 时停止, 问序列期望长度 ∈Zp∗, p=109+7\in\Z_p^*,~p=10^9+7∈Zp∗​, p=109+7

解题思路

O(mlog⁡m)O(m\log m)O(mlogm) 的概率DP做法看官方题解即可, 这里讲讲 1⩽m⩽1011,1⩽p⩽10121\leqslant m\leqslant 10^{11}, 1\leqslant p\leqslant 10^{12}1⩽m⩽1011,1⩽p⩽1012 时候 (即 第六届团体程序设计天梯赛 的 L3-3) 该怎么做

之后就很显然了, 我们做个杜教筛+整除分块就行了

u1s1, 这题考察的范围还挺广

可以加如下常数优化

  • 预先算一遍 ∑i=1mμ(i)\sum_{i=1}^m\mu(i)∑i=1m​μ(i), 这样整除分块过程中要求的所有 ∑μ\sum\mu∑μ 就都被求过一遍了
  • 因为我们往往使用 map 存 ∑μ\sum\mu∑μ, 所以在整除分块的过程中可以用变量记录上一步的 ∑μ\sum\mu∑μ

复杂度

假设

  • 预先筛出 μ(i), i=1,2,...,n\mu(i),~i=1,2,...,nμ(i), i=1,2,...,n
  • f(x)=⌊mx⌋f(x)=\lfloor{m\over x}\rfloorf(x)=⌊xm​⌋
  • 杜教筛求 ∑i=1nμ(i)\sum_{i=1}^n\mu(i)∑i=1n​μ(i) 的时间复杂度为 Θ(T(n))\Theta(T(n))Θ(T(n))
